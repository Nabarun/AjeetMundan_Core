App Overview

What is Ionic Framework?

Ionic is a powerful HTML5 native app development framework that helps you build native-feeling mobile apps all with web technologies like HTML, CSS, and Javascript.

Ionic is focused mainly on the look and feel, and UI interaction of your app. That means it is not a replacement for PhoneGap or your favorite Javascript framework. Instead, Ionic simply fits in well with these projects in order to simplify one big part of your app: the front end. We recommend reading Where does the Ionic Framework fit in? to get a good understanding of the framework's goals.

Ionic currently requires AngularJS in order to work at its full potential. While you can still use the CSS portion of the framework, you'll miss out on powerful UI interactions, gestures, animations, and other things.

To learn more about Ionic, please click here.

2. Install the app

After you have installed Ionic in your computer you are almost ready to start working on this app, but we need to install the app dependencies:

  1. Unzip the folder you have downloaded from Envato
  2. Move the unzipped folder to your preferred path inside your computer
  3. Open your terminal and go to your selected path
  4. Run npm install in order to install all the node modules required by the app

  5. Go to 'src/app/config.ts' file of the app root folder. This file contains firebase database cofiguration form where all the data is being fetched for this app.

    You must add your own firebase project configuration. To add your configuration go to firebase console, Open your project > click on overview > Add Firebase to your web app after that you can see the configuration.

  6. Run ionic serve to start a local dev server and watch/compile Sass to CSS
  7. To genarate resources run ionic resources.


3. Build Process


General Install Steps

  1. Install homebrew and node js via http://www.johnpapa.net/how-to-use-npm-global-without-sudo-on-osx/
  2. Install NPM package Gulp: npm install -g gulp

Android Setup Steps

  1. Install JDK 7
    • Download from http://www.oracle.com/technetwork/java/javase/downloads/jdk7-downloads-1880260.html
    • After install from the terminal create ~/.bash_profile if it doesnâ€™t already exist. You can use touch ~/.bash_profile to create the file.
    • Open the '.bash_profile' file you just created in any editor and add the following line:

      export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.7.0_72.jdk/Contents/Home

  2. Ant
    • Download from http://ant.apache.org/bindownload.cgi
    • Unzip to /users/[Your Username]/Development/ into a apache-ant-1.9.4
    • Open ~/.bash_profile and add the following line:

      export ANT_HOME=/users/jpjames/Development/apache-ant-1.9.4

  3. Android SDK
    • Download from http://developer.android.com/sdk/index.html#Other
    • Unzip to /users/[Your username]/Development
    • Open ~/.bash_profile and add the following line:

      export PATH=${PATH}:/users/[Your UserName]/Development/android-sdk-macosx/tools:/users/[Your Username]/Development/android-sdk-macosx/platform-tools:${ANT_HOME}/bin

  4. Download correct Android API
    • in terminal type source ~/.bash_profile to load the ~/.bash_profile changes.
    • type android from terminal. If everything is working correct it will open up the Android SDK Manager.
    • Install API 22 with SDK Platforms and Arm Image checked
  5. Configure Android Emulator
    • In terminal type android avd. If everything is working correct it will open up the Android AVD Manager.
    • Click the create button and add at least 1 device to emulate


iOS Setup Steps

  1. Install Xcode from app store.
    Once install is completed, open xcode and accept the license
  2. Install the IOS Simulator that Ionic will use:

    npm install -g ios-sim



Verifying that everything works

  1. Open terminal
  2. Navigate to the directory where you have saved the app project
  3. Type into the terminal: ionic info
    • It will show you the status of all the components required for ionic setup and build processes.
    • If there are any components missing, it will display that missing component and will also mention how to install that component.
  4. Type into the terminal: ionic cordova platform add android
  5. Type into the terminal: ionic cordova platform add ios
  6. Copy all dir in resources/ic_notification and place in platform/android/res
  7. Type into the terminal: ionic cordova build android
  8. Type into the terminal: ionic cordova build ios
  9. If running on a device run the command: ionic cordova run android (for Android device)
  10. If running on a device run the command: ionic cordova run ios (for iOS device)

Views and Controllers

If you take a look at the file structure overview in the previous section above, you will notice that the files of the different views/pages of the app are located in the path 'src/pages' directory of the project root folder.

The respective controllers of those views/pages are located in the same directory.

All routes of the pages are available in 'src/app/app.component.ts' file of the project root folder.

The syntax of the link is like :


import { Component, ViewChild } from '@angular/core';
import { Platform, MenuController, Nav } from 'ionic-angular';
import { StatusBar } from '@ionic-native/status-bar';
import { SplashScreen } from '@ionic-native/splash-screen';

import { PageService } from "../pages/page/page.service";
import { PageModel } from "../pages/page/page.model";
import { PushService } from './push.service';

@Component({
  templateUrl: 'app.html',
  providers: [StatusBar, SplashScreen]
})
export class MyApp {
  @ViewChild(Nav) nav: Nav;

  rootPage: any = 'home';
  pages: Array<{ title: string, component: any, icon: string, id?: string }>;
  activeMenu: string;
  currentMenu: string;
  pageList: Array;

  constructor(
    public platform: Platform,
    public menu: MenuController,
    private statusBar: StatusBar,
    private splashScreen: SplashScreen,
    public pageDb: PageService,
    private pushservice: PushService
  ) {
    this.initializeApp();

    this.pages = [
      { title: 'Home', component: 'home', icon: "std-icon-hairdresser-chair-1" },
      { title: 'Appointment', component: 'appointment', icon: "std-icon-calendar" },
      { title: 'Deals/Offers', component: 'deals', icon: "std-icon-perfume" },
      { title: 'Specialist', component: 'team', icon: "std-icon-male-hair" },
      { title: 'ServicePage', component: 'service', icon: "std-icon-barber" }
    ];

    if(this.platform.is('cordova')) {
      this.pushservice.pushInt();
    }
  }

  initializeApp() {
    this.platform.ready().then(() => {
      this.statusBar.styleDefault();
      this.splashScreen.hide();
    });
    this.listPages();
  }

  listPages() {
    this.pageDb.all().subscribe(res => {
      this.pageList = res;
    })
  }

  openPage(page) {
    this.menu.close();
      this.nav.push(page.component);
  }
  openStaticPage(page) {
    this.menu.close();
      this.nav.push('page', { id: page })
  }
}

Page Structure

There are a few types of typescripts used in this app:

  • Component
  • Service
  • Module
  • Model
  • Html
  • Scss

These files are located in 'src/pages' directory of the app root folder.

The file contents of the above folder are as shown below:

1. Component


The Component typescript files contain the respective ts of their respective html in the app.

For example, if we take a look at the file 'team.ts', there are some functions defined in this file for its respective view/page 'team.html', like allTeamMembers() to show all team members.

Similarly, each controller has defined functions for its respective views. If you take a look in these files, you should be able to understand the code of the functions in the files (provided you have a basic understanding of Angular and Javascript).

You can have a look at this link for more knowledge on Ionic and its components.


2. Service


Services can be accessed by any controller throughout the app. All that is required is for that service to be injected in the controller for the controller to access the functions of that service.

All pages have individual services like team page has team.service.ts file. This file contains all functions regarding data retrieval from API endpoint URL defined in 'config.ts' file.


3. Module


A module is a mechanism to group components, directives, pipes and services that are related, in such a way that can be combined with other modules to create an application.

Like team.module.ts file has declearation code for TeamPage component also has code for service providers and so on.


4. Model


An interface is an abstract type, it does not contain any code as a class does. It only defines the 'signature' or shape of an API. During transpilation, an interface will not generate any code, it is only used by Typescript for type checking during development.

If you check team.model.ts you can see that there are some code which defines the signature.


5. Html


The html file of the a particular page folder contaitn the design. This is simple html file which render the dynamic data.

If you check team.html you can see .


6. Scss


The scss file is used to design respective component. This is simple sass file. You can see team.scss file as a code example.
